# Нефункциональные требования (Non-Functional Requirements)

**Проект:** Мобильное приложение «Вертикаль» (Клиент)  
**Версия:** 1.0  

---

## Таблица NFR

| ID | Категория | Требование | Приоритет | Источник |
|----|-----------|------------|-----------|----------|
| NFR-001 | Платформа | Приложение должно работать на iOS 14.0+ и Android 8.0+. | Must | R-013 |
| NFR-002 | Платформа | Приложение должно быть разработано на React Native (единая кодовая база). | Must | R-013 |
| NFR-003 | Производительность | Время отклика UI (Time to Interactive) для основных экранов (список слотов, детали) должно быть < 2 секунд (p95) при 4G. | Must | Аналитик |
| NFR-004 | Производительность | API-запросы к бэкенду должны иметь таймаут 10 секунд. | Must | Аналитик |
| NFR-005 | Надёжность | Приложение должно корректно обрабатывать потерю соединения: показывать кэшированные данные для чтения, блокировать мутации. | Must | R-014 |
| NFR-006 | Безопасность | Все сетевые запросы должны осуществляться только по TLS 1.2+. | Must | Аналитик |
| NFR-007 | Безопасность | Сессия должна управляться через пару JWT (access + refresh). Access токен хранится в защищённом хранилище (Keychain/Keystore). | Must | Аналитик |
| NFR-008 | Безопасность | Запрос OTP должен быть ограничен rate-limit: не более 5 запросов на один номер телефона в час. | Must | NFR-SEC |
| NFR-009 | Безопасность | Персональные данные (номер телефона) не должны логироваться в клиентском приложении. | Must | Аналитик |
| NFR-010 | Доступность (A11y) | Интерфейс должен соответствовать стандартам WCAG 2.1 AA (контрастность, поддержка Dynamic Type, VoiceOver/TalkBack). | Should | Аналитик |
| NFR-011 | Уведомления | Push-уведомления должны доставляться через Firebase Cloud Messaging (Android) и APNs (iOS). | Must | R-007 |
| NFR-012 | Ограничения проекта | MVP должен быть готов к релизу в App Store и Google Play через 8 недель. | Must | R-017 |
| NFR-013 | Интеграции | Приложение должно интегрироваться только с существующим API скалодрома (бэкенд как black-box). | Must | R-004 |
| NFR-014 | Офлайн | Приложение должно кэшировать список слотов на последние 24 часа для просмотра без интернета. Мутации (бронь, отмена) запрещены в offline. | Should | R-014 |
| NFR-015 | Нагрузка | Клиентское приложение должно поддерживать пиковую нагрузку до 500 одновременных пользователей (чтение расписания в понедельник утром). | Must | Аналитик |
