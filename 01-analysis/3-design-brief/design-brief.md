# Дизайн-бриф: Мобильное приложение «Вертикаль»

**Проект:** Клиентское приложение для скалодрома «Вертикаль»  
**Версия:** 1.0  
**Дата:** 2026-07-04  
**Статус:** В работе

---

## 1. Цель и контекст продукта

**Продукт:** Мобильное приложение для самостоятельной записи клиентов на групповые тренировки по скалолазанию.  
**Проблема:** Ручная запись через Telegram приводит к овербукингу, потере клиентов и высокой нагрузке на администратора.  
**Решение:** Удобный мобильный интерфейс для просмотра расписания, бронирования слотов (с учётом проката снаряжения) и управления своими записями.

**Ключевые метрики UX:**
- Время от открытия приложения до подтверждения брони: < 2 минут.
- Процент успешных бронирований (без ошибок): > 95%.
- Снижение no-show за счёт напоминаний и прозрачного UI.

---

## 2. Роли пользователей

В рамках MVP приложение используется только одной ролью:
- **Клиент** — посетитель скалодрома, который ищет тренировку, записывается, отменяет бронь и оставляет отзывы.

*(Интерфейсы инструктора и администратора существуют в отдельной веб-админке и в скоуп мобильного приложения не входят).*

## 5. Навигация

**Структура:** Tab Bar (Нижняя панель)
1. **Расписание** (SCR-002) — главный экран по умолчанию.
2. **Мои записи** (SCR-005) — активные и прошедшие брони.
3. **Профиль** (SCR-007) — настройки и статистика.

**Переходы:**
- Из списка слотов (SCR-002) → в детали слота (SCR-003) по тапу на карточку.
- Из деталей слота (SCR-003) → в оформление брони (SCR-004) по CTA «Записаться».
- Из оформления (SCR-004) → модалка успеха (BS-002) → возврат на «Мои записи» (SCR-005).
- Из «Моих записей» (SCR-005) → в детали брони (SCR-006) по тапу.
- Из деталей брони (SCR-006) → модалка отмены (BS-003) или модалка карты (BS-004).
