# Модель данных (Ресурсная модель API)

**Проект:** Мобильное приложение «Вертикаль» (Клиент)  
**Версия:** 1.0  
**Дата:** 2026-07-04

> **Важно:** Данная модель описывает ресурсы API (то, что клиент читает и создаёт), а не схему базы данных. 
> Сущности, помеченные как `[READ-ONLY]`, поступают из существующего бэкенда скалодрома и не могут быть изменены клиентом.

---

## 1. Сущность: Slot (Слот / Тренировка)

**Назначение:** Представляет запланированную групповую тренировку. Клиент просматривает слоты для выбора и бронирования.  
**Доступ:** `[READ-ONLY]` (формируется бэкендом).

| Атрибут | Тип | Описание |
|---------|-----|----------|
| `id` | UUID | Уникальный идентификатор слота. |
| `start_time` | ISO8601 (UTC) | Время начала тренировки. На клиенте конвертируется в локальный часовой пояс. |
| `duration_minutes` | Integer | Длительность в минутах (обычно 90). |
| `format` | Enum | Формат тренировки: `BOULDERING` (болдеринг), `ROPE` (трассы с верёвкой). |
| `instructor` | Object (Instructor) | Агрегированный объект инструктора (см. ниже). |
| `capacity` | Integer | Максимальное количество мест в группе. |
| `booked_count` | Integer | Текущее количество забронированных мест. |
| `rental_gear` | Object (RentalGear) | Агрегированный объект доступности проката (см. ниже). |
| `base_price` | Decimal | Стоимость тренировки (без проката). Приходит из API, не хардкодится. |
| `address` | String | Адрес скалодрома для этой тренировки. |
| `status` | Enum | Статус слота: `SCHEDULED`, `CANCELLED_BY_ORGANIZER`. |

### Вложенный объект: Instructor
| Атрибут | Тип | Описание |
|---------|-----|----------|
| `id` | UUID | ID инструктора. |
| `name` | String | Имя и фамилия. |
| `photo_url` | String (URL) | Ссылка на фото. |
| `rating` | Decimal | Средний рейтинг (1.0 - 5.0). |

### Вложенный объект: RentalGear
| Атрибут | Тип | Описание |
|---------|-----|----------|
| `is_available` | Boolean | Есть ли свободные комплекты снаряжения на этот слот. |
| `shoes_price` | Decimal | Стоимость проката скальников. |
| `harness_price` | Decimal | Стоимость проката страховочной системы. |
| `available_sizes` | Array[Int] | Список доступных размеров скальников (например, `[40, 41, 43]`). Если массив пуст — проката нет. |

## 2. Сущность: Booking (Бронь)

**Назначение:** Запись клиента на конкретный слот. Создаётся и отменяется клиентом.  
**Доступ:** Read/Write (с ограничениями).

| Атрибут | Тип | Описание |
|---------|-----|----------|
| `id` | UUID | Уникальный идентификатор брони. |
| `slot_id` | UUID | Ссылка на слот. |
| `client_id` | UUID | Ссылка на клиента (совпадает с ID авторизованного пользователя). |
| `status` | Enum | Статус брони (см. переходы состояний ниже). |
| `rental_gear_needed` | Boolean | Заказал ли клиент прокат снаряжения. |
| `shoe_size` | Integer (Nullable) | Размер скальников, если `rental_gear_needed = true`. |
| `total_price` | Decimal | Итоговая стоимость к оплате на месте. Рассчитывается по формуле. |
| `created_at` | ISO8601 (UTC) | Время создания брони. |
| `cancelled_at` | ISO8601 (UTC, Nullable) | Время отмены брони. |
| `cancellation_reason` | String (Nullable) | Причина отмены (заполняется, если отменил скалодром). |

### Формула расчёта цены (Инвариант)
`total_price = slot.base_price + (rental_gear_needed ? (slot.rental_gear.shoes_price + slot.rental_gear.harness_price) : 0)`
> **R-003:** Оплата производится на месте. В MVP нет онлайн-транзакций, поле `total_price` носит информационный характер.

### Переходы состояний (Status Transitions)
[CONFIRMED] ──(клиент отменил до дедлайна)──> [CANCELLED]
[CONFIRMED] ──(клиент отменил после дедлайна)──> [LATE_CANCELLED]
[CONFIRMED] ─(скалодром отменил)──> [CANCELLED_BY_ORGANIZER]
[CONFIRMED] ──(тренировка прошла)──> [COMPLETED]
[CANCELLED_BY_ORGANIZER] ──(клиент выбрал новый слот)──> [RESCHEDULED] (опционально, Phase 2)

## 3. Сущность: Client (Клиент)

**Назначение:** Профиль пользователя приложения.  
**Доступ:** Read (свой профиль), Write (ограниченно, например, имя).

| Атрибут | Тип | Описание |
|---------|-----|----------|
| `id` | UUID | Уникальный идентификатор клиента. |
| `phone` | String | Номер телефона (E.164 формат). |
| `name` | String | Имя клиента (опционально заполняется после регистрации). |
| `is_regular` | Boolean | Статус «Постоянный клиент» (true, если посещений ≥ 5). |
| `late_cancellations_count` | Integer | Счётчик поздних отмен. |
| `no_shows_count` | Integer | Счётчик неявок. |
| `blocked_until` | ISO8601 (UTC, Nullable) | Дата и время окончания блокировки. Если `null` или в прошлом — клиент активен. |

> **R-005:** Блокировка наступает при 3 поздних отменах или 2 неявках. Длительность — 7 дней (для отмен) или 3 дня (для неявок).

---

## 4. Сущность: Review (Отзыв)

**Назначение:** Оценка инструктора клиентом после завершённой тренировки.  
**Доступ:** Write (создание), Read (просмотр в агрегате Instructor).

| Атрибут | Тип | Описание |
|---------|-----|----------|
| `id` | UUID | Уникальный идентификатор отзыва. |
| `booking_id` | UUID | Ссылка на завершённую бронь. |
| `instructor_id` | UUID | ID оценённого инструктора. |
| `rating` | Integer | Оценка от 1 до 5. |
| `comment` | String (Nullable) | Текстовый комментарий (макс. 500 символов). |
| `created_at` | ISO8601 (UTC) | Время создания. |

> **R-010:** Отзыв можно оставить в течение 24 часов после `slot.start_time + slot.duration_minutes`.

---

## 5. Общие правила и соглашения

1. **Время:** Все временные метки на сервере и в API хранятся и передаются в формате UTC (ISO8601). Клиентское приложение отвечает за конвертацию в локальный часовой пояс пользователя для отображения.
2. **Пагинация:** Для списков (например, история броней) используется курсорная или offset-пагинация: `limit` (default 20, max 50) и `offset`.
3. **Обработка отсутствующих данных:** Если поле опционально (Nullable), клиент должен корректно отображать fallback (например, прочерк или скрытие блока UI).
4. **Идемпотентность:** Все мутационные запросы (POST, DELETE) должны поддерживать заголовок `Idempotency-Key` на стороне API. Клиент генерирует UUIDv4 для каждого уникального действия.
