## 2. Обзор

**Назначение:** Экран авторизации клиента по номеру телефона через OTP-код. Обеспечивает быстрый вход без паролей и email-подтверждений.

**User Story:** US-001 — Как незарегистрированный пользователь, я хочу войти в приложение по номеру телефона, чтобы быстро начать пользоваться сервисом без создания пароля.

**Бизнес-ценность:** Минимальный порог входа увеличивает конверсию регистраций. OTP-авторизация снижает нагрузку на поддержку (не нужно восстанавливать пароли).

## 4. Входные данные

Экран не получает входных данных от других экранов. Работает автономно.

## 6. Инициализация

flowchart TD
    A[Открытие экрана] --> B{Есть сохранённые токены?}
    B -->|Да| C[Проверка access_token]
    C --> D{Валиден?}
    D -->|Да| E[Переход на SCR-002]
    D -->|Нет| F[Попытка refresh]
    F --> G{Успех?}
    G -->|Да| E
    G -->|Нет| H[Показ формы входа]
    B -->|Нет| H
    H --> I[Клиент вводит номер]
    I --> J[Запрос OTP]
    J --> K[Экран ввода кода]

## 7. Используемые запросы

### 7.1. Запрос OTP

+--------------+-------------------------------------------+
| Параметр     | Значение                                  |
+==============+===========================================+
| Метод        | POST                                      |
+--------------+-------------------------------------------+
| Путь         | /api/v1/auth/otp                          |
+--------------+-------------------------------------------+
| OperationId  | requestOtp                                |
+--------------+-------------------------------------------+
| Body         | { "phone": "+79001234567" }               |
+--------------+-------------------------------------------+
| Headers      | Content-Type: application/json            |
+--------------+-------------------------------------------+

**Обработка ответа:**

+------------------+---------------------------+---------------------------------------------------+
| Результат        | Условие                   | UI-реакция                                        |
+==================+===========================+===================================================+
| Успех            | 200 OK { otp_id, ttl }    | Переход к полю ввода OTP, запуск таймера 60 сек   |
+------------------+---------------------------+---------------------------------------------------+
| Неверный формат  | 400 { code: INVALID_PHONE}| Inline-ошибка под полем: «Проверь номер телефона» |
+------------------+---------------------------+---------------------------------------------------+
| Rate limit       | 429 { retry_after: 900 }  | Блокировка экрана на 15 мин с таймером            |
+------------------+---------------------------+---------------------------------------------------+
| Ошибка сети      | Timeout / 5xx             | Тост: «Не удалось отправить код. Проверь интернет»|
+------------------+---------------------------+---------------------------------------------------+

### 7.2. Проверка OTP

+--------------+-------------------------------------------+
| Параметр     | Значение                                  |
+==============+===========================================+
| Метод        | POST                                      |
+--------------+-------------------------------------------+
| Путь         | /api/v1/auth/verify                       |
+--------------+-------------------------------------------+
| OperationId  | verifyOtp                                 |
+--------------+-------------------------------------------+
| Body         | { "otp_id": "...", "code": "123456" }     |
+--------------+-------------------------------------------+

**Обработка ответа:**

+------------------+-------------------------------------------+---------------------------------------------------+
| Результат        | Условие                                   | UI-реакция                                        |
+==================+===========================================+===================================================+
| Успех            | 200 OK { access_token, refresh_token }    | Сохранение токенов → переход на SCR-002           |
+------------------+-------------------------------------------+---------------------------------------------------+
| Неверный код     | 401 { code: INVALID_OTP, attempts_left }  | Inline-ошибка: «Код не подошёл. Осталось: N»      |
+------------------+-------------------------------------------+---------------------------------------------------+
| Код истёк        | 410 { code: OTP_EXPIRED }                 | Тост: «Код устарел» → возврат к вводу номера      |
+------------------+-------------------------------------------+---------------------------------------------------+
| Ошибка сети      | Timeout / 5xx                             | Тост: «Ошибка соединения»                         |
+------------------+-------------------------------------------+---------------------------------------------------+

## 13. Критерии приёмки

**AC-001: Успешная авторизация**
- **Given:** Клиент открыл приложение впервые
- **When:** Ввёл валидный номер телефона и корректный OTP-код
- **Then:** Токены сохранены в Keychain/Keystore, выполнен переход на SCR-002

**AC-002: Неверный код**
- **Given:** Клиент на экране ввода OTP
- **When:** Ввёл неверный 6-значный код
- **Then:** Показана inline-ошибка «Код не подошёл. Осталось попыток: N», поле очищено

**AC-003: Rate-limit**
- **Given:** Клиент запросил OTP 5 раз за час
- **When:** Пытается запросить 6-й раз
- **Then:** Экран заблокирован на 15 минут с обратным отсчётом

**AC-004: Автопереход между ячейками OTP**
- **Given:** Клиент на экране ввода OTP
- **When:** Ввёл цифру в ячейку
- **Then:** Фокус автоматически перешёл на следующую ячейку

**AC-005: Повторная отправка кода**
- **Given:** Таймер повторной отправки истёк (60 сек)
- **When:** Клиент нажал «Отправить код снова»
- **Then:** Новый OTP отправлен, таймер сброшен
