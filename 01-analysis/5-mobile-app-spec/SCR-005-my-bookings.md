## 2. Обзор

**Назначение:** Отображение списка бронирований клиента с разделением на активные (предстоящие) и прошедшие. Позволяет быстро перейти к управлению конкретной записью.

**User Story:** US-009 — Как клиент, я хочу видеть историю своих посещений и активные брони, чтобы отслеживать планы и прогресс.

**Бизнес-ценность:** Централизованное управление записями снижает нагрузку на администратора (клиенты сами отменяют брони) и повышает retention за счёт отображения статистики.

## 10. Состояния экрана
Состояние	Условие	UI
Loading	Загрузка списка	Скелетоны карточек
Content	Данные загружены	Список карточек в активном табе
Empty (Активные)	Нет активных броней	Иллюстрация + «У тебя нет предстоящих тренировок»
Empty (Прошедшие)	Нет истории	«Ты ещё не был на тренировках»
Error	Ошибка сети, нет кэша	«Не удалось загрузить записи»

## 13. Критерии приёмки
AC-001: Группировка броней
Given: У клиента есть 2 активные и 3 прошедшие брони
When: Клиент открыл экран
Then: По умолчанию открыт таб «Активные» (2), в табе «Прошедшие» — 3 записи.
AC-002: Empty state для активных
Given: У клиента нет активных броней
When: Клиент открыл таб «Активные»
Then: Показана иллюстрация и кнопка «Посмотреть расписание».
AC-003: Отображение статуса
Given: Бронь отменена скалодромом
When: Клиент смотрит карточку
Then: Бейдж красный, текст «Отменена скалодромом».
