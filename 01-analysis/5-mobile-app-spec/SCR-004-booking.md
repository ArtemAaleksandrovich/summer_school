## 2. Обзор

**Назначение:** Экран выбора опций проката снаряжения и подтверждения стоимости перед созданием брони. Последний шаг перед фиксацией записи.

**User Story:** US-004, US-005 — Как клиент, я хочу записаться на тренировку и выбрать размер скальников, чтобы гарантированно получить место в группе и подходящую обувь.

**Бизнес-ценность:** Учёт прокатного снаряжения на этапе бронирования предотвращает ситуации, когда клиент приходит в зал, а нужного размера нет в наличии.

## 6. Инициализация

flowchart TD
    A[Открытие с slot_id] --> B[Запрос деталей слота]
    B --> C{Слот доступен?}
    C -->|Да| D[Показ формы оформления]
    C -->|Нет| E{Слот заполнен?}
    E -->|Да| F[Возврат на SCR-003 + тост Места закончились]
    E -->|Нет| G[Error state]
    D --> H[Выбор снаряжения клиентом]
    H --> I[Расчёт цены]
    I --> J[Подтверждение]

## 7. Используемые запросы
## 7.1. Создание брони

Параметр	Значение
Метод	POST
Путь	/api/v1/bookings
OperationId	createBooking
Headers	Idempotency-Key: {UUID}, Content-Type: application/json
Body	{ slot_id, rental_gear_needed, shoe_size? }

## Обработка ответа:

Результат	Условие
Успех	201 Created { booking }
Мест нет	409 { code: SLOT_FULL }
Нет размера	422 { code: RENTAL_GEAR_UNAVAILABLE }
Дедлайн пройден	400 { code: BOOKING_DEADLINE_PASSED }
Клиент заблокирован	403 { code: CLIENT_BLOCKED, blocked_until }

## 13. Критерии приёмки
AC-001: Выбор снаряжения
Given: Клиент на экране оформления
When: Выбрал «Взять в прокат»
Then: Появился выбор размера, цена увеличилась на сумму проката
AC-002: Недоступный размер
Given: Размер 42 занят (нет в available_sizes от API)
When: Клиент видит SizePicker
Then: Размер 42 серый, неактивен, пометка «Занят»
AC-003: Успешное бронирование
Given: Клиент выбрал снаряжение и нажал «Подтвердить бронь»
When: API вернул 201 Created
Then: Переход на BS-002, кнопка была заблокирована во время запроса
AC-004: Защита от дублей (Idempotency)
Given: Клиент быстро нажал «Подтвердить бронь» дважды
When: Оба тапа произошли за 200ms
Then: Отправлен только один запрос с тем же Idempotency-Key
AC-005: Ошибка — места закончились
Given: Места в слоте закончились во время оформления
When: API вернул 409 SLOT_FULL
Then: Показан тост «Места закончились, пока вы оформляли бронь», возврат на SCR-002
AC-006: Ошибка — нет нужного размера
Given: Выбранный размер закончился на сервере
When: API вернул 422 RENTAL_GEAR_UNAVAILABLE
Then: Alert «Размер 42 закончился. Попробуйте выбрать другой или без проката»
