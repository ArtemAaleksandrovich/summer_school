## 2. Обзор

**Назначение:** Детальный просмотр информации о конкретной тренировке перед принятием решения о бронировании. Показывает все детали: инструктора, формат, доступность мест и проката, адрес.

**User Story:** US-002 — Как клиент, я хочу видеть детали тренировки, чтобы принять решение о записи.

**Бизнес-ценность:** Полная прозрачность информации снижает количество отмен после бронирования. Отображение рейтинга инструктора помогает клиентам выбрать подходящего тренера.

## 7. Используемые запросы
## 7.1. Получение деталей слота

Параметр	Значение
Метод	GET
Путь	/api/v1/slots/{slot_id}
OperationId	getSlotById
Path params	slot_id: UUID

## Обработка ответа:

Результат	Условие	UI-реакция
Успех	200 OK { slot }	Показ всех деталей слота
Слот не найден	410 Gone	Возврат на SCR-002 + тост «Слот больше не существует»
Ошибка сети	Timeout / 5xx	Error state с кнопкой «Повторить»
401 Unauthorized	Токен истёк	Переход на SCR-001

## 13. Критерии приёмки
AC-001: Отображение всех деталей слота
Given: Клиент тапнул на карточку слота в списке
When: Детали успешно загружены
Then: Показаны: дата/время, формат, инструктор с рейтингом, доступность мест, информация о прокате, адрес
AC-002: Кнопка записи заблокирована за 30 минут
Given: До начала тренировки осталось 25 минут
When: Клиент открыл детали слота
Then: Кнопка «Записаться» серая, неактивна, текст «Запись закрыта»
AC-003: Открытие карты
Given: Клиент на экране деталей слота
When: Тапнул на «Как добраться»
Then: Открылась BS-004 с адресом и ссылками на карты
AC-004: Обработка удалённого слота
Given: Слот был отменён скалодромом после того, как клиент открыл его
When: Запрос деталей вернул 410
Then: Автоматический возврат на SCR-002 с тостом «Слот больше не существует»
AC-005: Отображение рейтинга инструктора
Given: У инструктора есть отзывы
When: Клиент смотрит детали слота
Then: Показан рейтинг (например, ⭐ 4.8) и количество отзывов
