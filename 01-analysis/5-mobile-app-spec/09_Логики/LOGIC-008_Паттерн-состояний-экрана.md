## 5. Описание логики (Конечный автомат)

Экран всегда находится в одном из 4 состояний. Приоритет состояний (от высшего к низшему): **Error > Loading > Empty > Content**.

1. **Loading:** `is_loading == true` и `data == null`.
   - *UI:* Скелетоны (shimmer). Не использовать полноэкранные спиннеры, если это список.
2. **Error:** `error != null` и `data == null`.
   - *UI:* Иконка ошибки, текст «Не удалось загрузить данные», кнопка «Повторить» (retry).
   - *Offline:* Если `is_offline == true` и есть кэш, показывать кэш + тост «Режим офлайн». Если кэша нет — полноценный Error.
3. **Empty:** `data != null` и `data.length == 0` (для списков).
   - *UI:* Иллюстрация, мотивирующий текст, кнопка действия (например, «Перейти к расписанию»).
4. **Content:** `data != null` и `data.length > 0` (или объект заполнен).
   - *UI:* Отображение контента. Доступен Pull-to-refresh.

## 7. Критерии приёмки
- **Given:** Сеть отключена, кэш пуст. **When:** Открыт список слотов. **Then:** Показан Error state с кнопкой «Повторить».
- **Given:** Сеть отключена, кэш есть. **When:** Открыт список слотов. **Then:** Показан кэш, сверху тост «Режим офлайн».
- **Given:** API вернул 200 OK, но массив слотов пуст. **When:** Загрузка завершена. **Then:** Показан Empty state.
