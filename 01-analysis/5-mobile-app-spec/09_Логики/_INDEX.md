## 1. Реестр логик

| ID | Название | Приоритет | Статус | Краткое описание |
|----|----------|-----------|--------|------------------|
| **LOGIC-001** | OTP-авторизация | Must | Draft | Запрос OTP, проверка кода, управление сессией и токенами |
| **LOGIC-002** | Расчёт доступности | Must | Draft | Проверка свободных мест в группе и прокатного фонда |
| **LOGIC-003** | Расчёт цены брони | Must | Draft | Динамический расчёт итоговой стоимости (база + прокат) |
| **LOGIC-004** | Отмена (правило 2 часов) | Must | Draft | Проверка дедлайна отмены, начисление санкций |
| **LOGIC-005** | Фильтрация слотов | Must | Draft | Семантика OR/AND для фильтров расписания |
| **LOGIC-006** | Карта маршрута | Should | Draft | Формирование deep links для внешних карт |
| **LOGIC-007** | Запрос push-разрешения | Must | Draft | Flow запроса и обработки пермишенов на уведомления |
| **LOGIC-008** | Паттерн состояний экрана | Must | Draft | Универсальный конечный автомат экранов (Loading/Content/Empty/Error) |
