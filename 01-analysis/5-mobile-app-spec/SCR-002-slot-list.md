## 2. Обзор

**Назначение:** Главный экран приложения. Отображает расписание тренировок на ближайшие 7 дней с возможностью фильтрации по формату, дате и инструктору.

**User Story:** US-002 — Как клиент, я хочу видеть список тренировок на неделю, чтобы спланировать своё посещение скалодрома.

**Бизнес-ценность:** Прозрачное расписание повышает заполняемость групп. Удобная фильтрация ускоряет поиск подходящей тренировки.

## 7. Используемые запросы
## 7.1. Получение списка слотов

Параметр	Значение
Метод	GET
Путь	/api/v1/slots
OperationId	getSlots
Query params	date_from (ISO8601), date_to (ISO8601), format (enum[]), instructor_id (uuid[]), limit (default 20)

## Обработка ответа:

Результат	Условие	UI-реакция
Успех	200 OK { slots: [...], has_more }	Показ списка карточек, кэширование
Пустой список	200 OK { slots: [] }	Empty state: «На эти дни тренировок пока нет»
Ошибка сети	Timeout / 5xx	Если есть кэш → показ кэша + тост; если нет → Error state
401 Unauthorized	Токен истёк	Переход на SCR-001

## 13. Критерии приёмки

AC-001: Отображение списка слотов
Given: Клиент открыл приложение и авторизовался
When: Загрузка слотов успешна
Then: Показан список карточек с временем, форматом, инструктором и количеством мест
AC-002: Фильтрация по дате
Given: Клиент на главном экране
When: Тапнул на чип «Ср 14»
Then: Показаны только слоты на 14 число, чип выделен
AC-003: Empty state
Given: На выбранные даты нет слотов
When: Клиент открыл экран
Then: Показан empty state с иллюстрацией и текстом «На эти дни тренировок пока нет»
AC-004: Offline-режим
Given: Клиент уже загружал расписание
When: Открыл приложение без интернета
Then: Показаны кэшированные слоты, тост «Режим офлайн»
AC-005: Pull-to-refresh
Given: Клиент на главном экране
When: Сделал свайп вниз
Then: Список обновился, показан индикатор загрузки
