# Описание домена — Скалодром «Вертикаль»

**Проект:** Мобильное приложение для записи на тренировки  
**Дата:** 2026-07-04  
**Статус:** Согласовано

---

## 1. Бизнес-контекст

### О компании

**Скалодром «Вертикаль»** — частный скалодром, расположенный в бывшем складском ангаре. Работает 3 года, специализируется на групповых тренировках по скалолазанию.

**Текущая ситуация:**
- Запись на тренировки ведётся вручную через Telegram и тетрадку
- Администратор (Оля) тратит 15+ минут на обработку одной записи
- Происходят двойные бронирования (овербукинг)
- Клиенты забывают о записях (no-show до 15% случаев)
- Нет возможности масштабировать бизнес

**Проблема:** Ручное управление записями неэффективно, приводит к конфликтам и потере клиентов.

**Решение:** Мобильное приложение для самостоятельной записи клиентов на групповые тренировки.

## 2. Основной бизнес-процесс

### Сценарий записи на тренировку
Скалодром формирует расписание на неделю вперёд
├── Время начала (каждые 1.5 часа)
├── Зона/формат (болдеринг или трассы с верёвкой)
├── Инструктор (один из 5 сотрудников)
└── Количество мест (до 16, для новичков — до 8)
Клиент открывает приложение
└── Видит список свободных слотов на 7 дней
Клиент выбирает слот
├── Фильтрует по формату/дате/инструктору
├── Видит доступные места
└── Видит информацию о прокате снаряжения
Клиент оформляет бронь
├── Указывает, нужен ли прокат снаряжения
├── Если да — выбирает размер скальников (36-45)
└── Подтверждает запись
Система проверяет доступность
├── Свободные места в группе
├── Свободный прокатный фонд (если нужен прокат)
└── Валидность времени (не позже чем за 30 мин до начала)
Бронь создаётся
├── Клиент получает подтверждение (push)
├── Напоминание за 2 часа до тренировки
└── Запись появляется в профиле клиента
Клиент приходит на тренировку
├── Показывает бронь инструктору
├── Получает снаряжение (если заказал прокат)
└── Оплачивает на месте (наличные/перевод)
После тренировки
└── Клиент может оценить инструктора (1-5 звёзд, комментарий)


### Альтернативные сценарии

**Отмена клиентом:**
- До дедлайна (за 2 часа): место освобождается, без последствий
- После дедлайна: место освобождается, но клиент получает предупреждение
- 3 поздних отмены → блокировка на 7 дней

**Отмена скалодромом:**
- Причина: профилактика, проверка снаряжения
- Все записавшиеся получают push-уведомление
- Возможность переноса на другой слот

**No-show (клиент не пришёл):**
- 2 no-show → блокировка на 3 дня

## 3. Цель приложения

### Основная цель
Снизить нагрузку на администратора, устранить овербукинг и no-show, повысить удовлетворённость клиентов.

| M-003 | No-show | 15% | 5% | BR-003 |

## 4. Ключевые объекты домена

| Объект | Определение | Атрибуты | Связи |
|--------|-------------|----------|-------|
| **Слот (Slot)** | Запланированная групповая тренировка | id, start_time, duration (1.5h), format (болдеринг/трассы), instructor_id, capacity, booked_count, rental_gear_available, address | 1:N → Booking, N:1 → Instructor |
| **Бронь (Booking)** | Запись клиента на слот | id, client_id, slot_id, status (подтверждена/отменена/завершена/отменена_скалодромом), rental_gear_needed, shoe_size, created_at, cancelled_at | N:1 → Slot, N:1 → Client |
| **Инструктор (Instructor)** | Сотрудник скалодрома | id, name, photo_url, rating, reviews_count | 1:N → Slot, 1:N → Review |
| **Клиент (Client)** | Пользователь приложения | id, phone, name, is_regular, late_cancellations_count, no_shows_count, blocked_until | 1:N → Booking, 1:N → Review |
| **Прокатный фонд (Rental Gear)** | Снаряжение для аренды | total_shoes (30), available_shoes, sizes (36-45), total_harnesses, available_harnesses | Связан со Slot |
| **Отзыв (Review)** | Оценка инструктора | id, client_id, instructor_id, rating (1-5), comment, created_at | N:1 → Client, N:1 → Instructor |

### Статусы брони

Подтверждена (Confirmed)
├── → Завершена (Completed) — клиент пришёл на тренировку
├── → Отменена (Cancelled) — клиент отменил до дедлайна
├── → Отменена_поздно (Late_Cancelled) — клиент отменил после дедлайна
└── → Отменена_скалодромом (Cancelled_By_Organizer) — отмена организатором
Завершена (Completed)
└── → Оценена (Reviewed) — клиент оставил отзыв
Отменена_скалодромом (Cancelled_By_Organizer)
└── → Перенесена (Rescheduled) — клиент выбрал новый слот

## 5. Пользователи и роли

| Роль | Описание | Цели | В скоупе MVP? |
|------|----------|------|---------------|
| **Клиент** | Посетитель скалодрома | Записаться на тренировку, отменить бронь, оценить инструктора | ✅ Да |
| **Инструктор** | Ведущий тренировок | Просмотр списка записавшихся | ❌ Нет (уже есть в админке) |
| **Администратор/Владелец** | Оля и сотрудники | Управление расписанием, просмотр статистики | ❌ Нет (уже есть в админке) |

### Клиент (единственная роль в скоупе MVP)

**Цели:**
1. Быстро найти подходящую тренировку
2. Записаться на слот (с прокатом или без)
3. Управлять своими бронями (просмотр, отмена)
4. Получить напоминание о тренировке
5. Оценить инструктора после тренировки

**Боли:**
- Долгая запись через Telegram
- Неясно, есть ли свободные места
- Забывает о записях
- Нет истории посещений

## 6. Ограничения

### Бизнес-ограничения
- **Бюджет:** ограниченный, нет средств на сложные интеграции
- **Срок:** 8 недель до начала сезона
- **Оплата:** только наличные/перевод на карту (нет онлайн-оплаты в MVP)
- **Масштаб:** один скалодром, нет сети

### Технические ограничения
- **Существующий бэкенд:** black-box, нельзя менять
- **Только клиентское приложение + API:** без админки (уже есть)
- **React Native:** для кроссплатформенности (iOS + Android)
- **Нет легаси-данных:** проект с нуля

### Проектные ограничения
- **MVP:** только роль Клиента
- **Язык:** только русский
- **Уведомления:** Push + SMS (без email)
- **Офлайн:** только просмотр расписания (read-only)

## 8. Глоссарий

| Термин | Определение | Пример |
|--------|-------------|--------|
| **Слот** | Временной интервал для групповой тренировки (1.5 часа) | "Слот в среду 19:00 — болдеринг с инструктором Иваном" |
| **Болдеринг** | Формат тренировки: лазание без страховки на небольшой высоте (до 4-5 метров) | Подходит для новичков, до 8 человек в группе |
| **Трассы с верёвкой** | Формат тренировки: лазание со страховкой на высоте (до 15 метров) | Для опытных, до 16 человек в группе |
| **Дедлайн отмены** | Время (2 часа до начала), до которого можно отменить без последствий | "Отменил за 3 часа — ок, за 1 час — предупреждение" |
| **No-show** | Ситуация, когда клиент не пришёл на тренировку без отмены | "Записался и не пришёл, не предупредив" |
| **Постоянный клиент** | Клиент с 5+ посещениями, имеет приоритет при записи | "5 раз был — получил метку 'постоянный'" |
| **Прокатный фонд** | Количество доступных комплектов снаряжения для аренды | "30 скальников, 20 страховочных систем" |
| **OTP** | One-Time Password — одноразовый код для авторизации по SMS | "Код 123456 действителен 5 минут" |
| **Овербукинг** | Двойное бронирование одного места | "Два клиента записались на одно место" |
| **Late cancellation** | Поздняя отмена (после дедлайна) | "Отменил за 1 час до начала" |
| **Idempotency-Key** | Заголовок запроса для предотвращения дублирования операций | "Ключ для защиты от двойной записи" |

## 9. Предположения и риски

### Предположения

1. **Существующий бэкенд стабилен** — API работает без сбоев, SLA 99.9%
2. **Клиенты готовы к мобильному приложению** — основная аудитория использует смартфоны
3. **Инструкторы готовы к оценкам** — нет сопротивления со стороны сотрудников
4. **Прокатный фонд достаточен** — 30 комплектов покрывают спрос
5. **Push-уведомления работают** — клиенты не отключают уведомления

### Риски

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| API бэкенда нестабилен | Средняя | Высокое | Кэширование на клиенте, retry-логика |
| Клиенты не используют приложение | Низкая | Высокое | Упрощённый UX, обучение на месте |
| Инструкторы не готовы к оценкам | Средняя | Среднее | Анонимность отзывов, модерация |
| Прокатного снаряжения не хватает | Низкая | Среднее | Ограничение количества броней с прокатом |
| Клиенты отключают push-уведомления | Высокая | Среднее | Дублирование SMS для критичных уведомлений |
