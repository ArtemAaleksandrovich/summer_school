## 3. Тест-кейсы

### Задача 1: Авторизация (SCR-001)

---

#### TC-001: Успешная авторизация с валидным номером и OTP

**Приоритет:** High  
**Предусловия:**
- Приложение установлено и запущено
- Mock API работает
- Нет активной сессии (токены удалены)

**Шаги:**
1. Открыть приложение
2. На экране SCR-001 ввести номер телефона: `+79001234567`
3. Нажать кнопку «Получить код»
4. Дождаться появления экрана ввода OTP (6 ячеек)
5. Ввести код: `123456`
6. Дождаться автоматического подтверждения или нажать «Подтвердить»

**Ожидаемый результат:**
- Шаг 2: Поле ввода отображает номер в формате `+7 (900) 123-45-67`
- Шаг 3: Кнопка переходит в состояние loading (спиннер)
- Шаг 4: Появляется экран с 6 ячейками для OTP, запускается таймер 60 сек
- Шаг 5: При вводе 6-й цифры автоматически отправляется запрос на верификацию
- Шаг 6: Происходит переход на главный экран (SCR-002, вкладка «Расписание»)
- В консоли появляется лог: `[MOCK] OTP для +79001234567: 123456`
- Токены сохраняются в SecureStore

**Фактический результат:**  
**Статус:**  
**Связанные требования:** FR-001, FR-002, FR-004, US-001, SCR-001

#### TC-002: Автопереход между ячейками OTP

**Приоритет:** Medium  
**Предусловия:**
- Пользователь на экране ввода OTP
- OTP уже отправлен

**Шаги:**
1. Ввести первую цифру кода (например, `1`)
2. Ввести вторую цифру (`2`)
3. Ввести третью цифру (`3`)
4. Продолжить ввод до 6-й цифры

**Ожидаемый результат:**
- После ввода каждой цифры фокус автоматически переходит на следующую ячейку
- Все 6 ячеек заполняются последовательно
- После ввода 6-й цифры автоматически отправляется запрос на верификацию

**Фактический результат:**  
**Статус:**  
**Связанные требования:** FR-002, SCR-001

#### TC-013: Отображение карточек слотов с полной информацией

**Приоритет:** High  
**Предусловия:**
- Список слотов загружен

1. Проверить отображение каждой карточки слота
2. Убедиться в наличии всех элементов

**Ожидаемый результат:**
Каждая карточка содержит:
- Время начала (крупно, формат ЧЧ:ММ)
- Бейдж формата («Болдеринг» — синий, «Трассы» — оранжевый)
- Аватар инструктора + имя + рейтинг (⭐ X.X)
- Количество свободных мест («Свободно X из Y»)
- Иконку проката 👟 (если доступен)
- При малом количестве мест (≤3) — оранжевый текст «Мало мест!»

**Фактический результат:**  
**Статус:**  
**Связанные требования:** FR-009, SCR-002

## 5. Примечания

### Тестовые данные
- **OTP-код:** `123456` (фиксированный для всех запросов)
- **Номер телефона:** любой валидный, например `+79001234567`
- **Инструкторы:** 5 моковых (Иван Петров, Анна Сидорова, Дмитрий Кузнецов, Елена Морозова, Алексей Волков)
- **Слоты:** ~20-25 на 7 дней, форматы BOULDERING и ROPE

### Известные ограничения
1. Mock API не реализует все сценарии ошибок (например, 403 CLIENT_BLOCKED)
2. Нет реального бэкенда для проверки интеграции
3. Offline-режим не тестировался (требуется дополнительная настройка)

### Рекомендации
- Перед запуском тестов убедиться, что mock API работает корректно
- Для тестирования ошибок временно модифицировать `handlers.ts`
- Использовать React Native Debugger для инспекции состояния
